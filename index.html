Problem Statement

Traditional web hosting requires manual server configuration, ongoing maintenance, and manual scaling. These factors increase complexity, cost, and the risk of misconfiguration.

Proposed Solution

This project demonstrates a serverless approach using AWS Amplify to host a static web application without managing physical or virtual servers.

System Architecture

The architecture leverages fully managed AWS services to ensure scalability, security, and global availability.

  • GitHub Source Repository
  • Automated CI/CD Pipeline
  • AWS Amplify Hosting
  • Amazon CloudFront CDN

Technologies Used

The following technologies were selected to minimize complexity while following modern cloud best practices.

  • HTML5
  • CSS3
  • JavaScript
  • AWS Amplify
  • HTTPS / TLS
  • AWS Free Tier

Deployment Workflow

The deployment process is automated and follows DevOps principles.

  1. Code is pushed to a GitHub repository
  2. AWS Amplify detects changes
  3. The site is built and deployed automatically
  4. Content is served globally via CloudFront

Security & Reliability

Security and availability are handled by AWS managed services, reducing operational overhead.

  • HTTPS Enabled by Default
  • No Server Maintenance
  • Global Edge Network

Final Outcome

The completed application is:

  • Scalable
  • Secure
  • Cost-efficient
  • Deployed in under two hours

Conclusion

This project demonstrates how modern cloud platforms like AWS enable rapid deployment of reliable web applications using serverless architectures.
